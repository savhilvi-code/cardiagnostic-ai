# Frontend Codex Rules

- Перед изменениями читать `ARCHITECTURE_FRONTEND.md`.
- Перед изменениями проверять `FRONTEND_TASK_LOG.md`.
- Не менять backend из frontend-задач.
- Не менять API URL без объяснения.
- Не ломать существующую навигацию страниц.
- Не удалять существующие HTML/CSS/JS-файлы без причины.
- Все изменения делать маленькими шагами.
- После правки объяснять, какие файлы изменены и зачем.
- Если задача непонятна, сначала задать вопрос.
- Не делать массовый редизайн без отдельного разрешения.
